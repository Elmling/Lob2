Avatars.cs

API

Class Description

author: Elm

description - Avatars Class
requires: https:github.com/Elmling/Avatar_API
Allows for users to define simple avatars
to be used with the Avatar API to present
and easy to use avatar system - instead
of trying to work with the default system.

Method 1

avatars::update(%this)

Notes: Updates all avatars to be current in the system

Method 2

avatars::displayBegin(%this,%client)

Notes: using the avatar class we start to display the avatars to a client allowing them to cycle through available avatars, allowing them to choose Uses Menu.cs

Method 3

avatars::displayEnd(%this,%client)

Notes: stops displaying the avatar menu and the cycling

Method 4

avatars::displayNext(%this,%client)

Notes: displays the next available avatar to the client

Method 5

avatars::displayPrevious(%this,%client)

Notes: displays the previous available avatar to the client

Method 6

avatars::setAvatar(%this,%client,%avatar)

Notes: vatar api defines a client attribute so we dont need to do anything. Ex: %client.customAvatar => current avatar

Method 7

avatars::getAvatarAPIName(%this,%name)

Notes: support for AvatarAPI

Method 8

GameConnection::onDeath(%this, %sourcePlayer, %sourceClient, %damageType, %damageArea)

Notes: Packaged - support for smmdeadbodies when using AvatarAPI
